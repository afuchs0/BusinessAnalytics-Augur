# Augur

## Quick start
Go in terminal at the location "website" and then tun

```bash
npm install
npm run dev
```
If it doesn't work simply copy paste the README.md file in any LLM for a quick fix... (it's probably because you don't have Node.js or brew. The llm will give you the 2 commands to donwload it.)

Then open <http://localhost:3000>.

> **Read the signal. Back the founder.**
 
A demo web application for **Augur**, a pre–Series A success-probability tool for Swiss VC analysts. Built as an interactive companion to the ETH Zürich D-MTEC Business Analytics (363-1098-00L, FS26) Group 1 final report.

This is **not** a production system. It is a high-fidelity prototype that visualises the ML pipeline, feature importance, and analyst workflow described in the report. We heavily used Claude AI to build this website.

## Stack

- **Next.js 14** (App Router) + **React 18** + **TypeScript**
- **Tailwind CSS** for styling, with a custom *ink / signal / bone* palette
- **Framer Motion** for staggered reveals and counters
- **Recharts** for the feature-importance chart
- **lucide-react** for iconography

Fonts: **Fraunces** (display), **Geist** (sans), **Geist Mono** (mono).

## Data

All startup data is **synthetic** and generated deterministically in `lib/data.ts` from a seeded PRNG. The schema mirrors the variables used in the report's gradient-boosted model: `category_code`, `pre_series_a_team_size`, `pre_series_a_clevel`, `pre_series_a_employees`, `people_linked_total`, `avg_team_edu_score`, `share_people_with_any_stem_degree`, `people_with_degree_data`, `round_1`, `duration_1`, etc.

The feature importance values shown on `/about` and `/startup/[id]` come directly from **Figure 3a** of the report.

The four practitioner quotes on `/about` are drawn verbatim from the interview appendix (Pavel Yakovlev — Tenity, Andrea Fritschi — Tenity CIO, Flo Pattiwael — Photon-Ventures, Bernhard Petermeier — XISTA).

## Design notes

- **ML output** and **analyst notes** are rendered with deliberately different visual languages. ML cards use the cyan *signal* palette and a data-grid aesthetic; the analyst notes panel uses warm *bone* tones and an explicit "not generated by the model" banner. This mirrors the human-in-the-loop framing in the report.
- The pipeline table uses `FORWARDED` / `FILTERED` rather than "Success" / "Failure" badges, reflecting Augur's role as a **triage layer**, not a decision engine.
- Dark-mode only.
